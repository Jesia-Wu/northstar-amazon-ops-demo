# 002：私有静态试用站规格

## 1. 目标

把已验收的 Amazon ASIN 运营工作台初级版本发布成一个可直接打开的静态试用站，供当前项目方先试用。发布不改变任何运营功能、数据、账号或外部连接边界。

## 2. 冻结范围

- 使用现有 Vite + React 前端及固定内置演示资料。
- 产出可被 Sites 托管的静态资源与最小 Worker 入口；SPA 深链接应回退至 `index.html`。
- 创建一个 Sites 项目、保存一个源代码版本，并优先部署为仅项目所有者可访问的私有试用地址。
- 仅使用用户提供的冰水背景和当前仓库已有前端资源。

## 3. 明确不做

- 不公开发布、不绑定自定义域名、不增加登录、数据库、埋点或第三方脚本。
- 不接 MCP、Amazon、LLM、真实账号、真实数据或外部写入。
- 不修改现有运营工作台的功能、视觉或业务状态模型。

## 4. 风险与访问边界

- 风险等级：高风险；原因是创建外部站点、推送源代码并产生生产部署地址。
- 访问策略：仅在 Sites 能验证为 owner-only 时使用私有部署；如果平台要求公开或共享访问，停止并请求用户明确批准。
- 资料边界：站点只含内置演示资料；用户提供的背景图仅用于本次私有试用，公开发布前必须再次确认图像与包装文字的使用权。
- 回滚：不覆盖本地已验证版本；若私有站加载异常，不执行公开部署，保留已保存版本并以新验证版本重新部署。

## 5. 冻结接口与状态归属

| 项目 | 冻结决定 | 状态拥有者 |
| --- | --- | --- |
| 运营工作台状态 | 保持浏览器内存及固定 fixture | 现有前端 |
| 托管入口 | 仅静态资源与 SPA 回退 | Sites Worker 入口 |
| 项目凭据 | Sites 短期写入凭据，仅命令内使用 | 托管平台 |
| 访问控制 | owner-only 私有试用 | Sites |

## 6. 验收标准

| ID | 验收标准 |
| --- | --- |
| AC-201 | 生产构建通过，并包含 Sites 所需的静态资源和 Worker 入口。 |
| AC-202 | 部署源代码与构建包指向同一提交；站点元数据只保存项目 ID，不保存凭据。 |
| AC-203 | 仅在平台确认 owner-only 私有访问时完成部署；否则不做公开部署。 |
| AC-204 | 部署状态为成功，并返回可直接打开的试用地址。 |
| AC-205 | 已发布版本继续满足第一期的无 MCP、无 Amazon、无 LLM、无真实数据边界。 |
