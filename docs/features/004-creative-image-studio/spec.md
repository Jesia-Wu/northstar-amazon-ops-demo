# 004：卖点驱动图片创作台（本地原型）规格

## 目标

把当前“创意资产”页面从静态图片墙升级为运营可试用的创作工作台：运营填写产品名称和卖点，选择图片用途与视觉方向，得到一组可编辑的图片方案和模拟生成队列。它服务于 Amazon Listing / A+ 的图片规划，而不是通用 AI 画图聊天框。

## 用户与核心路径

目标用户是准备新品图片、希望把“卖点”变成可给设计师或图像模型执行的画面任务的 Amazon 运营。

1. 运营进入“创意资产”。
2. 填写产品名称，提供 1–4 条卖点，选择图片用途与画面方向。
3. 点击“生成视觉提案”。
4. 页面在本地展示生成中状态，随后给出四张结构化的图片方案：画面目的、构图、要表达的卖点、可直接交给模型的提示词摘要。
5. 运营点击任一方案，审阅其画面说明并继续修改卖点后重新生成。

## 本期范围（冻结）

- 桌面端 `>= 1024px` 的“创意资产”工作台重构。
- 产品名称、卖点、图片用途、画面方向、尺寸与禁用元素等本地表单状态。
- 四种可选视觉方案；选择后更新大预览、卖点映射和提示词摘要。
- “生成视觉提案”本地模拟：有禁用、进行、成功、空/校验错误状态；固定使用本地演示素材，不发出网络请求。
- 清楚标注“本地创意草稿，不会上传产品信息或调用图像模型”。
- 用户可复制提示词摘要；复制失败必须显示可恢复说明。
- 使用既有冰水背景与 Summer Glass Operations 的冰白、雾蓝、薄荷系统；不采用通用紫色 AI 渐变。

## 明确不做

- 不接 GPT Image、OpenAI、MCP、Amazon 或任何第三方接口。
- 不让用户在浏览器输入、保存、展示或传输 API Key。
- 不上传产品图片，不生成真实图片，不持久化草稿，不导出文件。
- 不做移动端专项适配、多人协作、版本历史、费用计量或批量生成。

## 后续真实生成服务边界（非本期实现）

真实服务另建高风险功能包。浏览器只可调用同源受控接口：

```ts
type ImageGenerationRequest = {
  productName: string;
  sellingPoints: string[];
  channel: 'listing' | 'a-plus';
  directionId: string;
  aspectRatio: '1:1' | '4:5' | '16:9';
  avoid: string;
};
```

- API Key 仅由服务端环境变量持有；不得进入 React 状态、静态包、浏览器存储、日志或 Git 提交。
- 服务端校验输入、限流、记录无敏感内容的请求 ID 与状态，并返回受限的生成任务结果。
- 用户提供的卖点属于业务输入；接入前需明确保存期限、用途和数据处理边界。

## 状态与接口归属（冻结）

| 状态 | 拥有者 | 行为 |
| --- | --- | --- |
| `CreativeBriefDraft` | `CreativeStudio` React 组件 | 仅浏览器内存；刷新可丢失 |
| `CreativeDirection` | `demoCreativeStudio` fixture | 固定本地演示数据 |
| `GenerationStatus` | `CreativeStudio` React 组件 | `idle → generating → ready`；模拟完成后显示方案 |
| 图片预览 | 静态本地素材 | 仅表达样机，不宣称模型已生成 |
| API Key / 服务端任务 | 本期不存在 | 不适用 |

## 验收标准

| ID | 验收点 |
| --- | --- |
| AC-001 | 创意资产页首先呈现产品卖点输入、视觉方向和生成队列，不再只显示静态图库。 |
| AC-002 | 空产品名称或没有有效卖点时，禁止生成并显示中文可恢复说明。 |
| AC-003 | 填入有效产品名与卖点后，按钮先进入“正在编排视觉方案”，再进入“4 个视觉提案已就绪”。 |
| AC-004 | 四个提案都显示图片用途、解决的购买问题、卖点映射和提示词摘要；点击提案会更新主预览。 |
| AC-005 | 所有生成结果明确标为本地演示方案；页面没有 API Key 输入、没有外部网络请求。 |
| AC-006 | 提示词摘要可复制，复制成功或失败均有文字反馈。 |
| AC-007 | 1440px 桌面端三栏工作流可读，无横向溢出；键盘可抵达输入、方向选择、生成和复制操作。 |
| AC-008 | `prefers-reduced-motion` 下没有持续旋转或大位移动画；生成状态仍以文本可读方式呈现。 |

## 变更控制

接入真实 GPT Image、添加图片上传、持久化草稿或发布生成图片，都要先新建功能规格并重新评估为高风险；不得把本地模拟按钮悄悄替换成真实外部调用。
